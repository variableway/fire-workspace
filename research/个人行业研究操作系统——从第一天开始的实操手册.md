# 个人行业研究操作系统：从第一天开始的实操手册

> 停止阅读理论。本文只包含**可执行的步骤、可直接使用的模板、能验证结果的方法**。选一个领域，从今天开始，按步骤执行。
>
> 目标：30天内建立一套可运转的个人研究系统，产出第一份经过验证的洞察。

---

## 第0天：启动（30分钟）

### Step 1：选定一个领域（5分钟）

不要多选。**只选一个**你最关心、最常用到、或最可能产生实际收益的领域。

**决策标准**：
- 这个领域的变化会影响你的决策（消费、投资、职业、社交）
- 你愿意每天至少花15分钟看它
- 你有一定基础认知（不是完全陌生）

**如果犹豫，从以下选一个**：
- 时尚消费品（已有完整指南）
- 科技/AI（变化快，信息多）
- 二级市场（股票/基金/ crypto）
- 你所在的行业（职业相关）

**输出**：在备忘录写下：**"我的研究域：[具体名称]"**

---

### Step 2：建立工作区（10分钟）

创建三个固定位置：

| 位置 | 工具 | 用途 |
|------|------|------|
| **收件箱** | 手机备忘录/Flomo/微信收藏 | 随时 dumping，不整理 |
| **工作区** | Notion/飞书文档 | 概念整理、结构化笔记 |
| **数据库** | Notion Database/飞书多维表格 | 时序数据、对比表格 |

**立即操作**：
1. 打开手机备忘录，新建一个笔记，命名为"[领域]收件箱"
2. 打开Notion/飞书，新建一个页面，命名为"[领域]研究"
3. 在该页面下新建一个Database/多维表格，命名为"[领域]数据追踪"

---

### Step 3：确定你的第一个问题（15分钟）

不要研究"这个领域的一切"。**从一个具体问题开始**。

**好问题的标准**：
- 有明确的答案形式（是/否、数字、排名、时间）
- 在30天内可以验证
- 对你有实际意义

**示例问题**：
- "SK-II神仙水在未来3个月会涨价还是降价？"
- "Nike Dunk系列的热度是否在衰退？"
- "AI编程工具中，Cursor和GitHub Copilot哪个在开发者中增长更快？"
- "我关注的那只基金，当前估值处于历史什么位置？"

**输出**：在工作区顶部写下你的第一个问题，并标注**验证日期**（今天+30天）。

---

## 第1-7天：第一周——建立扫描习惯

### 每日15分钟固定动作

**早晨（或任意固定时间），设一个15分钟闹钟，执行以下动作**：

```
┌─────────────────────────────────────────┐
│  第1步：扫描（5分钟）                     │
│  打开3个你选定的信号源，快速浏览            │
│  记录任何与你的问题相关的信息               │
│  只记录，不分析，不判断                    │
├─────────────────────────────────────────┤
│  第2步：捕获（5分钟）                     │
│  把有价值的信息 copy 到"收件箱"            │
│  格式：日期 + 来源 + 关键事实（一句话）     │
├─────────────────────────────────────────┤
│  第3步：标记（5分钟）                     │
│  给每条信息打一个标签：                    │
│  [价格]/[热度]/[政策]/[新品]/[负面]/[存疑]  │
└─────────────────────────────────────────┘
```

### 第一天：寻找你的3-5个信号源

今天的主要任务是**找到你的初始信号源**。

**操作方法**：
1. 打开Google/微信搜一搜/小红书，搜索"[你的领域] 资讯"
2. 看前20个结果，打开你觉得"看起来专业"的5-8个
3. 每个看30秒，判断：
   - 更新频率？（日更/周更/月更？）
   - 信息质量？（有数据？有来源？还是纯观点？）
   - 是否独立？（是不是某个品牌的官方号？）
4. 选出**3个**你感觉最舒服、信息密度最高的

**记录到工作区**：创建一个页面"信号源清单"，写下：

| 名称 | 类型 | 更新频率 | 信息质量 | 独立性 | 备注 |
|------|------|---------|---------|--------|------|
| 例：腕表之家 | 论坛 | 实时 | 高 | 高 | 表友一手信息 |
| 例：Hodinkee | 媒体 | 日更 | 高 | 高 | 国际视角 |
| 例：得物 | 交易平台 | 实时 | 中 | 中 | 价格信号 |

---

### 第二天到第七天：建立肌肉记忆

每天重复15分钟动作。**不要跳过，不要加量**。

**本周结束时的检查清单**：
- [ ] 连续7天完成了15分钟扫描
- [ ] 收件箱里有20-50条原始记录
- [ ] 信号源清单有3个活跃的、你每天都在看的来源
- [ ] 你开始感觉到"这个领域最近发生了什么"

**如果做不到每天15分钟，停止。** 说明这个领域不是你真正关心的，换一个。

---

## 第2-4周：建立系统——从收件箱到工作区

### 第8天：第一次整理（1小时）

打开你的收件箱，50条左右的记录。今天做一次结构化整理。

**操作步骤**：

**Step 1：分类（20分钟）**

在工作区创建以下页面/分区：
- **概念**：这个领域的基础术语、框架、规则
- **玩家**：关键品牌/公司/人物，及其动态
- **数据**：价格、销量、搜索指数等可量化信息
- **事件**：重要发布、政策、争议
- **我的判断**：你的预测、假设、待验证的想法

把收件箱里的50条记录，分到这5个区。**一条记录只进一个区。**

**Step 2：提取（20分钟）**

在每个区里，问自己：
- "这10条记录中，有没有3-5条指向同一个趋势？"
- "有没有互相矛盾的信息？"
- "有没有让我意外的事实？"

把发现的**模式**写在区的顶部。**模式比单条记录重要100倍。**

**Step 3：形成假设（20分钟）**

基于你发现的模式，回答你的初始问题。

格式：
```
我的当前判断：[是/否/方向]
支持证据：
  1. [来源A的事实]
  2. [来源B的事实]
  3. [来源C的事实]
反对证据：
  1. [什么可能让我错？]
  2. [我还不知道什么？]
验证方式：[到X日期，如果Y发生，则判断正确；如果Z发生，则判断错误]
```

**把这个假设写下来。这是你的第一份"洞察草稿"。**

---

### 第9-14天：建立数据库

每天15分钟动作升级：

```
第1步：扫描（5分钟）—— 同前
第2步：捕获（3分钟）—— 只捕获与"当前假设"相关的
第3步：更新（7分钟）—— 把关键数据录入数据库
```

**数据库最小结构**（在Notion/飞书表格中创建）：

**表1：时序数据**
```
日期 | 指标名称 | 数值 | 来源 | 备注
例：2024-01-15 | Rolex水鬼二手价 | ¥12.5万 | Chrono24 | 较上周+3%
```

**表2：事件日志**
```
日期 | 事件 | 影响评估 | 相关假设 | 验证状态
例：2024-01-10 | 劳力士宣布涨价8% | 短期推高二手价 | 假设A | 待验证
```

**表3：假设追踪**
```
假设ID | 假设内容 | 提出日期 | 验证日期 | 当前状态 | 结果 | 复盘
例：H1 | 水鬼二手价Q1上涨 | 1月15日 | 4月15日 | 进行中 | - | -
```

**每天只更新1-2行。不要贪多。**

---

### 第15-21天：引入交叉验证

升级你的信号源。**现在应该有3个在跑的，再加2个不同类型的。**

**交叉验证矩阵**：

| 你的假设 | 信号源A（内容） | 信号源B（交易） | 信号源C（搜索） | 结论 |
|---------|---------------|---------------|---------------|------|
| 例：水鬼涨价 | 表友讨论热烈 | 二手价确实上涨 | 百度指数上升 | ✅ 三源一致 |

**每天扫描时，特别关注："今天看到的信息，是支持还是反对我的假设？"**

**如果发现强烈反对证据，不要忽视。** 在工作区创建一个"反方证据"区，专门记录这些。

---

### 第22-28天：准备输出

你的假设已经运行了2-3周。现在准备第一次验证。

**操作**：
1. 打开你的"假设追踪"表
2. 看每个假设的验证日期是否到了
3. 对比现实与预测
4. 在工作区写一篇"验证报告"，格式：

```
# 假设验证报告 #[编号]

## 原始假设
[复制你的假设]

## 预测条件
[如果X发生，则假设正确]

## 实际结果
[发生了什么]

## 判断
[正确 / 错误 / 部分正确]

## 为什么对/错
[分析原因，不是找借口]

## 我忽略了什么
[诚实面对盲点]

## 更新后的假设
[基于验证，调整或提出新假设]
```

## 第29-30天：第一次复盘

花2小时，做第一次系统复盘。

### 复盘清单

**系统层面**：
- [ ] 我的3-5个信号源中，哪个最有效？哪个最无效？
- [ ] 我是否花了太多时间在低价值信息上？
- [ ] 数据库是否好维护？还是需要简化？

**认知层面**：
- [ ] 我对这个领域的理解，比30天前深了多少？
- [ ] 我最初的假设，有多少被验证了？
- [ ] 我犯了什么系统性错误？（过早下结论？忽视反方证据？）

**行动层面**：
- [ ] 这些洞察是否指导了我的任何实际决策？
- [ ] 如果没有，为什么？（是洞察不够强？还是我没行动？）

### 输出物

**一份"月度研究简报"**，包含：
1. 本月关键发现（3-5条）
2. 验证结果（假设对/错的分析）
3. 下月关注重点（2-3个新问题）
4. 系统改进（信号源调整、流程优化）

**这份简报可以**：
- 给自己看（倒逼思考）
- 给朋友看（获取反馈）
- 发到社交媒体（建立个人品牌）

## 可复用的工具模板

### 模板1：每日扫描记录（手机备忘录）

```
[日期] [领域]扫描
─────────────────
来源1：[名称]
  - [记录1，一句话]
  - [记录2，一句话]
来源2：[名称]
  - [记录1]
来源3：[名称]
  - [记录1]
─────────────────
今日模式：
  [如果有，写一句；如果没有，写"未发现明显模式"]
待跟进：
  [有什么需要明天查的？]
```

**实际示例**：
```
2024-01-15 手表扫描
─────────────────
来源1：腕表之家
  - 劳力士2024年涨价清单流出，平均涨幅8%
  - 表友反应两极：囤货的庆祝，等跌的骂娘
来源2：Chrono24
  - 水鬼价格周涨幅+3%，日志涨幅+1.5%
来源3：小红书
  - "第一块劳力士选什么"笔记量激增
─────────────────
今日模式：
  涨价消息在内容端快速发酵，但交易端反应滞后（+3% vs 官宣8%）
待跟进：
  明天看得物/国内二手商是否跟进涨价
```

### 模板4：AI辅助Prompt（严格限定使用场景）

**场景1：快速扫盲**
```
我对[XX概念]完全不了解。请用300字解释：
1. 它是什么
2. 为什么重要
3. 在这个领域，懂它和不懂它的人有什么区别

注意：如果有具体数据（年份、金额、人名），请明确标注"[需人工验证]"。
```

**场景2：结构化整理**
```
以下是我收集的关于[XX话题]的10条信息：
[粘贴]

请帮我：
1. 按主题分类
2. 指出其中互相矛盾的信息
3. 指出哪些信息需要进一步验证（缺乏来源或与其他来源冲突）

不要添加你自己的判断，只做整理和标记。
```

**场景3：反方论证**
```
我形成了以下观点：[你的观点]

请扮演这个领域最资深的怀疑论者，找出这个观点的3个最大漏洞。
要求：
- 每个漏洞附上一个具体的事实或逻辑问题
- 不要安慰我，尽量让我不舒服
```

**场景4：验证清单**
```
我基于[A事实]和[B事实]，得出了[C结论]。

请帮我生成一个验证清单：
1. 要证明这个结论成立，还需要什么证据？
2. 什么证据会推翻这个结论？
3. 这个推理中，哪个环节最薄弱？

注意：你不需要知道正确答案，只需要帮我列出验证方向。
```

## 常见失败模式与对策

| 失败模式 | 症状 | 对策 |
|----------|------|------|
| **扫描疲劳** | 每天15分钟变成负担，开始跳过 | 减少到10分钟；或换领域；或改为隔天扫描 |
| **收集癖** | 收件箱1000条，从未整理 | 设上限：收件箱超过50条就必须整理；或改为只记录"与当前假设相关"的 |
| **过早下结论** | 第3天就形成一个"洞察"，之后再无视反方证据 | 强制要求：任何假设必须运行至少2周才能验证；必须有"反对证据"区 |
| **AI依赖症** | 所有整理都让LLM做，自己不动脑 | 强制要求：所有洞察必须先手写草稿，再让AI润色；AI只辅助表达，不辅助思考 |
| **没有验证** | 提出了10个假设，从未回头看对错 | 日历设提醒；每月强制写验证报告 |
| **孤岛研究** | 从不与人交流，不知道自己的盲点 | 每月至少向1个人（朋友/网友/同事）展示你的研究，获取反馈 |
